

## 📋 Задача 1. Циклический сдвиг и Память

**Уровень:** Средний +  
**Тема:** Присваивание, ссылки, арифметика переменных

### Часть А: Тройной обмен
Даны три переменные: `a`, `b`, `c`.  
Необходимо выполнить циклический сдвиг значений:
- Значение `a` должно перейти в `b`
- Значение `b` должно перейти в `c`
- Значение `c` должно перейти в `a`

**Требование:**
1. Реализуйте это через промежуточную переменную `temp`
2. Реализуйте это в **одну строку** через кортежную распаковку (например: `a, b, c = ...`)
3. Выведите `id()` всех переменных до и после обмена. Совпали ли идентификаторы?

### Часть Б: Арифметический обмен
Даны две числовые переменные `x` и `y`.  
Поменяйте их значения местами **без использования третьей переменной** и без синтаксиса `a, b = b, a`.

> *Подсказка:* Используйте только сложение и вычитание (или умножение и деление)

**Пример работы:**
```text
--- Часть А ---
До: a=1, b=2, c=3
После: a=3, b=1, c=2

--- Часть Б ---
До: x=10, y=5
После: x=5, y=10
```

## 📋 Задача 2. Кассир 2.0 (Налоги и Округление)

**Уровень:** Продвинутый  
**Тема:** Типы данных (`float`/`int`), точность вычислений, форматирование

### Задание
Напишите программу расчета чека. Пользователь вводит:
1. Цену товара за единицу (может быть дробной)
2. Количество товаров (целое)
3. Ставку налога в процентах (например, `20` для 20%)

**Необходимо рассчитать и вывести:**
1. Сумму без налога (Цена × Количество)
2. Сумму налога (отдельной переменной)
3. Итоговую сумму к оплате
4. Сдачу с 5000 рублей
5. Максимальное количество таких товаров, которое можно купить на 5000 рублей (целое число, без использования условий)

**Требование:**
- Все денежные суммы должны выводиться ровно с **двумя знаками после запятой** (используйте f-строки: `{value:.2f}`)

**Пример работы:**
```text
Введите цену товара: 150.5
Введите количество: 3
Введите налог (%): 20

Сумма без налога: 451.50
Сумма налога: 90.30
Итого к оплате: 541.80
Внесено: 5000.00
Ваша сдача: 4458.20
На 5000 руб. можно купить: 27 шт. (без учета налога в этом расчете)
```

## 📋 Задача 3. Конвертер Времени PRO

**Уровень:** Эксперт  
**Тема:** Целочисленное деление, остаток от деления, сложные вычисления

### Часть А: Секунды в Дни
Пользователь вводит большое количество секунд (целое число).  
Разложите это время на составляющие и сохраните в **отдельные переменные**:
- `days` (дни)
- `hours` (часы)
- `minutes` (минуты)
- `seconds` (секунды)

> *Напоминание:* В сутках 24 часа, в часе 60 минут, в минуте 60 секунд

### Часть Б: Обратный конвертер
Пользователь вводит дни, часы, минуты и секунды (4 отдельных ввода).  
Посчитайте **общее количество секунд** и также рассчитайте, **какую часть суток** (в процентах) занимает это время.

**Требование:**
1. Используйте осмысленные имена переменных (не `d`, `h`, `m`, `s`, а `total_days`, `input_hours` и т.д.)
2. Процент дня должен выводиться с точностью до **4 знаков после запятой**

**Пример работы:**
```text
--- Часть А ---
Введите секунды: 90061
Прошло: 1 дн. 1 ч. 1 мин. 1 сек.

--- Часть Б ---
Введите дни: 0
Введите часы: 6
Введите минуты: 0
Введите секунды: 0
Всего секунд: 21600
Это занимает: 25.0000% от суток
```

## ✅ Критерии успеха

| Критерий | Описание |
|----------|----------|
| **Чистота кода** | Между логическими блоками кода есть пустые строки |
| **Имена переменных** | Нет переменных типа `a`, `b`, `c` (кроме задачи на обмен), все имена понятны (`price`, `tax_amount`, `total_seconds`) |
| **Типы данных** | Вы понимаете, где у вас `int`, а где `float`, и не теряете точность там, где это важно (деньги) |
| **Отсутствие лишнего** | Не использованы списки, условия и циклы |
